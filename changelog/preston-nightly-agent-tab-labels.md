Fixed the Caretaker's tab never opening on its own. The in-workspace chat UI decides which tabs to auto-open (and which agent is the hidden services agent) from each agent's labels, but the lightweight `mngr observe --discovery-only` stream the UI listens to does not carry labels -- so every agent arrived label-less and the Caretaker was never recognized as an auto-created agent. The system interface now reads each agent's labels from its local state on disk when the discovery stream omits them, so the Caretaker tab reliably pops up (and the services agent stays correctly hidden).
