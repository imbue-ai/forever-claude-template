Fixed the Docker image build, which broke when `setup_system.sh` started sourcing its sibling `scripts/_provision_guard.sh` (added in the Lima provisioning-skip-guard change) but the Dockerfile only copied `setup_system.sh` into `/usr/local/bin`. At build time the script resolved the guard via `dirname "$0"` to `/usr/local/bin/_provision_guard.sh`, which did not exist, so the `fct-setup-system` RUN failed with `No such file or directory`. The Dockerfile now copies `_provision_guard.sh` alongside `setup_system.sh` so the source resolves. The Lima path was unaffected because it runs the script in place from `scripts/`, where the guard already sits beside it.
