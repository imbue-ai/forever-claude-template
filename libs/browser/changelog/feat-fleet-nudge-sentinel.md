The fleet's agent-facing nudges (browser handed back, browser gone) are now wrapped in an `<agentic-browser-fleet>...</agentic-browser-fleet>` sentinel so the transcript UI renders them as a collapsed system chip instead of a bare user bubble.

`BrowserSession._message_agent` -- the path that messages a queued agent via `mngr message` when its browser frees up or disappears -- wraps its text via the new `_wrap_system_message` helper (`_SYSTEM_MESSAGE_TAG`). The wrapping lives here, in the fleet's own service; mngr is left untouched (it is an independent product with no stake in this display concern). The sentinel adds no newlines, so the wrapped message types into the agent's pane identically to before, and the agent still resumes its turn on receipt exactly as before -- this is display-only.
