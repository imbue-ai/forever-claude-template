- Fixed a first-boot deadlock that left a fresh credential-less workspace on a permanently dead "No events yet for this agent" chat. When the initial chat agent's `mngr create` failed after registering the agent (the `--message /welcome` delivery timed out at claude's no-credentials login screen), every subsequent boot retried the create and hit an instant duplicate-name collision, so the agent id needed by the welcome resend was never persisted. Bootstrap now looks for an existing active agent named after the host before creating; if exactly one exists it adopts it -- persisting its id and writing the run-once signal without re-sending `/welcome` (the welcome arrives via the auth-success resend once the user signs in). A failed or ambiguous lookup falls through to the plain create path.
