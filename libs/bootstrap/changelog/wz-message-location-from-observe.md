- The bootstrap now persists the initial chat agent's id so the welcome-resend can address it directly. `mngr create` for the initial chat agent runs with `--format json`; the bootstrap reads the created agent's id from that output and writes it to `$MNGR_HOST_DIR/initial_chat_agent_id`. system_interface's welcome-resend reads that file and messages the agent by its stable id instead of re-resolving it by name. A failed id parse is logged but still touches the creation signal, so the chat agent is never created twice.
