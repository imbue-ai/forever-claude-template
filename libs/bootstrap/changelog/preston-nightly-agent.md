Bootstrap now prepares the workspace for cron-scheduled jobs and the weekly Caretaker. Each boot it fetches the user's IANA timezone from the minds desktop client through the latchkey gateway (a small tenacity retry -- the gateway tunnel may not be up that early; the clock stays on UTC when the fetch fails, and the desktop client's "timezone unknown" answer is accepted immediately without retrying) and points `/etc/localtime` + `/etc/timezone` at it so schedules run in the user's local time.
