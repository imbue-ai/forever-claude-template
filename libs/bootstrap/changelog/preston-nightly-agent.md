Bootstrap now prepares the workspace for cron-scheduled jobs and the nightly Caretaker. Each boot it snapshots the agent environment to `/run/minds-agent-env` (0600; sourced by `scripts/with_agent_env.sh` because cron scrubs job environments), fetches the user's IANA timezone from the minds desktop client through the latchkey gateway (bounded retry -- the gateway tunnel may not be up that early; the clock stays on UTC when the fetch fails) and points `/etc/localtime` + `/etc/timezone` at it so schedules run in the user's local time, and creates `runtime/caretaker/` for the Caretaker's run logs and permissions file (runtime/ persists with the container volume, and rides GitHub sync when the user has enabled it).

Fixed: a first boot with no Claude credentials could deadlock the initial chat forever. `mngr create ... --message /welcome` timed out after the agent was already registered, so every later boot collided on the agent name. The bootstrap now looks up an existing agent named after the host before creating and adopts a survivor from a partial create: its id is persisted as the welcome-resend target and the create-once signal written, letting the auth-success resend deliver the welcome after sign-in.
