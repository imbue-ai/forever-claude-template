Bootstrap now prepares the workspace for cron-scheduled jobs and the nightly Caretaker. Each boot it snapshots the agent environment to `/run/minds-agent-env` (0600; sourced by `scripts/with_agent_env.sh` because cron scrubs job environments), fetches the user's IANA timezone from the minds desktop client through the latchkey gateway (bounded retry -- the gateway tunnel may not be up that early) and points `/etc/localtime` + `/etc/timezone` at it so schedules run in the user's local time, and creates `runtime/caretaker/` so the Caretaker's run logs and permissions file ride the runtime-backup branch.

The Caretaker never runs on creation day: on first boot the bootstrap seeds its daily-job stamp with today's local date, so the first run lands at the next day's 3 AM due hour. When the timezone cannot be fetched at first boot, a fixed-offset `Etc/GMT*` placeholder is adopted that puts the first run about 8 hours after setup. Seeding (and the placeholder) happen exactly once per workspace, tracked by a `caretaker.seeded` sidecar marker rather than the stamp's absence -- deleting the stamp to force a same-day run now survives a reboot without being re-seeded or having the clock re-placeheld.

Fixed: a first boot with no Claude credentials could deadlock the initial chat forever. `mngr create ... --message /welcome` timed out after the agent was already registered, so every later boot collided on the agent name. The bootstrap now looks up an existing agent named after the host before creating and adopts a survivor from a partial create: its id is persisted as the welcome-resend target and the create-once signal written, letting the auth-success resend deliver the welcome after sign-in.
