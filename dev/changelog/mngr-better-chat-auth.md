Add `scripts/migrate_claude_auth.py`, the manual migration for existing workspaces: it moves any Claude credentials (`ANTHROPIC_API_KEY`, `ANTHROPIC_BASE_URL`, `CLAUDE_CODE_OAUTH_TOKEN`) from the mngr host env file into the env block of the shared `CLAUDE_CONFIG_DIR/settings.json`, scrubs them from `$MNGR_HOST_DIR/env`, and restarts the workspace's claude agents. The restart phase runs detached, so an agent invoking the script on itself still completes the migration. Run from the repo root: `uv run python scripts/migrate_claude_auth.py`. Idempotent; subscription-based workspaces need no migration.

The vendored mngr snapshot is synced to pick up `mngr start --resume-message`, which the workspace's auth-change restart uses to restart agents and deliver the "please continue" message in one fused call.
