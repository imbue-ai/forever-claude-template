Commits an agent makes through the Bash tool are now attributed to that agent instead of to whatever git identity the workspace checkout inherited at create time (previously the operator's, e.g. "Josh Albrecht").

The PreToolUse hook that tags Bash subprocesses for memory shedding has been renamed from `scripts/claude_oom_tag_subprocess.py` to `scripts/claude_rewrite_bash_command.py` and now also prepends this agent's git commit identity to every Bash command: the author/committer name is the agent's current mngr name (read live, so a `mngr rename` is reflected on the next commit without restarting the agent), and the email is the agent's mngr routing address in id form (`<agent_id>@<host_id>`), which is stable across renames and still routes via `mngr message`. Set as `GIT_AUTHOR_*`/`GIT_COMMITTER_*` env vars, this wins over the checkout's git config without changing it. When run outside a mngr container (e.g. local dev), the identity is left to git's own resolution.
