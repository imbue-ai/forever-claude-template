- Messaging an agent (`POST /api/agents/{id}/message`) no longer runs a fresh mngr discovery on every send. `AgentManager` now retains each agent's location (host/provider) from the `mngr observe` discovery stream it already consumes -- rebuilt on every full-snapshot event and dropped on destroy events -- and exposes `get_agent_matches_by_name`. `agent_discovery.send_message` consults that live, event-fed state first and messages the agent directly via mngr's pre-resolved `agents_to_message` API; it falls back to a full discovery only when the location is unknown (e.g. a just-created agent before the next snapshot) or when the known location no longer reaches the agent. The result is fresh on both agent creation and removal (the observe stream updates it), with no staleness window for the common case, and discovery stays as a correctness backstop.
