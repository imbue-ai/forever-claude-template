- Messaging an agent (`POST /api/agents/{id}/message`) no longer runs a fresh mngr discovery on every send. `AgentManager` now retains each agent's location (host/provider) from the `mngr observe` discovery stream it already consumes -- rebuilt on every full-snapshot event, updated incrementally when an agent-discovered delta arrives, and dropped on destroy events -- and exposes `get_agent_matches_by_id`. `agent_discovery.send_message` consults that live, event-fed state first and messages the agent directly via mngr's pre-resolved `agents_to_message` API; it falls back to a full discovery only when the location is unknown or when the known location no longer reaches the agent. The result is fresh on both agent creation and removal, and discovery stays as a correctness backstop.
- A just-created agent is located the instant its agent-discovered delta arrives, not only at the next full snapshot, so the *first* message to a freshly-created agent also skips discovery. The location only needs the agent's `host_id` + `provider_name` (both on the delta), so every discovered agent is recorded immediately -- no dependency on the host being seen before.
- Messaging now addresses agents **by id only**. `send_message(agent_id: AgentId, ...)` no longer accepts a name, so a message always targets the exact, globally-unique agent rather than fanning out across same-named agents on other workspaces/hosts (which is never an intended behavior). The endpoint passes the id it already has (O(1) location lookup). The welcome-resend reads the initial chat agent's id that the bootstrap persisted at `$MNGR_HOST_DIR/initial_chat_agent_id` and sends by that id -- no name resolution at all, so the prior `host_name`-from-`data.json` lookup (and the fail-safe for a name spanning hosts) is gone.
- All agent messaging and resolution now go through `AgentManager`, the owner of the live cache. `send_message_to_agent(agent_id, message)` reads the manager's own location cache (no lookup is passed in); `get_agent_info_by_id` resolves an agent to its `AgentInfo` from the live state. The message endpoint (`_find_agent`) and the welcome-resend path (resolve + send) are both wired to these at startup, so they share one cache instead of each running its own discovery -- the welcome resend, previously the only path that re-discovered, now resolves *and* sends through the cache. `agent_discovery` is back to pure mngr-API calls (it takes the already-resolved location as a value).
