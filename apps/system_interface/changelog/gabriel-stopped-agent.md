
- Agent terminal start: opening an agent's terminal now starts the agent through the exact same in-process mngr path that sending it a message uses (`agent_discovery.start_agent`, which calls mngr's `ensure_agent_started` via `resolve_to_started_host_and_running_agent`), instead of shelling out to a fresh `mngr start` subprocess. Opening a terminal and messaging an agent now succeed or fail together rather than diverging on environment or config differences between the server process and the subprocess. mngr's own lifecycle check makes the start a no-op for an already-running agent.
