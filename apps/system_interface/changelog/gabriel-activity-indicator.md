
- Activity indicator: a chat agent that was restarted mid-turn (e.g. its container was restarted while it was working) no longer shows "Thinking..." forever. The indicator derives THINKING/TOOL_RUNNING from the transcript tail, which after a mid-turn kill stays pinned on an unmatched `tool_use` / trailing `tool_result` because the abandoned turn never emits its closing `assistant_message`. The activity tracker now compares the tail's timestamp against the mtime of mngr's `claude_process_started` marker (touched on every startup/resume): a tail older than the current Claude process belongs to a turn this process never ran, so it is shown IDLE rather than working. Genuinely in-flight turns are unaffected (their events postdate the process start), and a mid-turn auto-compaction does not move the boundary.
