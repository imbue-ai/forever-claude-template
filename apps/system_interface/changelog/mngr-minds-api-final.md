
- Claude sign-in: a failed Claude Code OAuth code submission no longer leaves you stuck on a dead "Verify & finish" form. A submitted code consumes the single-use OAuth session end-to-end (the backend terminates its `claude auth login` subprocess unconditionally, and the frontend clears its session id), so re-submitting on the same form silently did nothing -- no request, no error, no feedback. The login modal now routes any OAuth code failure to the full error screen whose only action is "Start over", which restarts sign-in from the beginning (spawning a fresh OAuth session) so you can actually try again. The header close button and backdrop still dismiss the modal. API-key submit failures are unchanged -- those stay retryable in place.
