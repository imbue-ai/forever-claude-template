Fixed stale image caching in the chat. Images referenced by chat markdown were served straight from their on-disk path with a one-year immutable cache policy, so overwriting a previously referenced file made new messages show the browser's stale cached copy, and re-rendering old messages from a cold cache showed the file's new content instead of what was originally posted.

Each message's images are now frozen per message: the frontend rewrites inline chat image URLs to `/api/chat-images/<event_id>/<path>`, and the backend snapshots the source file's bytes (content-hashed, deduplicated copies under the mngr host dir) the first time each (event, path) pair is seen -- eagerly when the session watcher emits a new message, and lazily on first fetch as a fallback. A new message always fetches the file's current content, and an already-posted message keeps showing exactly the bytes that existed when it was posted, even if the file on disk later changes.
