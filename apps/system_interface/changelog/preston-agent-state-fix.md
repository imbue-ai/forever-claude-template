
- Activity indicator: a chat agent's status now correctly reflects when the agent's Claude process has actually stopped, instead of staying stuck on "Thinking..."/"Running..." after e.g. a container restart. The `mngr observe --discovery-only` stream that feeds the agent list is metadata-only and reports every agent as RUNNING, so a dead agent's abandoned transcript tail kept deriving THINKING/TOOL_RUNNING forever. The system interface now runs a lightweight local poll (every few seconds, scoped to the agents whose tmux session lives in this same container) that probes each agent's real process liveness via a single `tmux list-panes` + `ps` snapshot and mngr's existing `determine_lifecycle_state`. When the poll finds an agent's process gone, the running gate closes and the indicator settles on IDLE; when it comes back, the transcript-derived state resumes. Genuinely in-flight agents are unaffected.

- Sending a chat message while the workspace backend is temporarily unreachable (e.g. the container is mid-restart after the laptop woke from sleep) no longer pops up a confusing "Failed to send message: null" alert and silently drops the message. The message is now held in the composer with a "Reconnecting…" indicator and is automatically re-sent as soon as the live connection recovers; the draft text is preserved throughout. Genuine application-level send failures still surface an explicit error alert as before, and a held message that can never be delivered (the backend stays gone) is given up on after a few minutes rather than lingering forever.
