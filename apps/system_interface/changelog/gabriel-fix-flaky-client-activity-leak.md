Fixed a flaky test (`test_context_round_trips_through_script_and_endpoint`) that intermittently failed CI on main. The WebSocket handler resolved the client-activity events path from process-global environment (`MNGR_HOST_DIR` / `MNGR_AGENT_ID`) at message-processing time. In tests several servers share one process's environment, so a client-activity write from a lingering WebSocket connection (e.g. a browser e2e test still tearing down) could land in a *different* server's activity log — which the layout `context` test then read as a client it never created. The handler now captures the primary agent's layout directory once, at connect time, and uses it for every write on that connection, so writes always target the connection's own workspace regardless of later environment mutation. No behavior change in production (a single server's environment is stable for its lifetime).
