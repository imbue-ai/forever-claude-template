- Per-agent liveness dot on chat tabs: each agent's dockview tab shows a colored dot reflecting its mngr lifecycle state, distinct from the chat's activity indicator (which tracks work within a running process). The dot is green while the agent's claude process is up and working (RUNNING), yellow while the process is up but idle and waiting on the user (WAITING), and grey while the agent is dormant -- its process is not running (DONE, STOPPED, REPLACED, UNKNOWN) but revives on the next message. Hovering shows the exact lifecycle state. The state is carried on the agent WebSocket payload's `state` field (surfaced from mngr's discovery stream, which probes whether the expected claude process is alive in the agent's tmux pane), so the dot turns grey within one observe cycle when a process dies -- including when the OOM handler (earlyoom) sheds it.
