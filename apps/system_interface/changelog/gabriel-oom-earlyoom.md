- Per-agent liveness dot on chat tabs: each agent's dockview tab shows a colored dot reflecting its mngr lifecycle state, distinct from the chat's activity indicator (which tracks work within a running process). The dot is green while the agent's claude process is up and working (RUNNING), yellow while the process is up but idle and waiting on the user (WAITING), and grey while the agent is dormant -- its process is not running (DONE, STOPPED, REPLACED, UNKNOWN) but revives on the next message. Hovering shows the exact lifecycle state. The state comes from a lightweight per-agent lifecycle poll the system interface runs every few seconds -- it lists the workspace's local agents and reads each one's probed lifecycle state (mngr probes whether the expected claude process is alive in the agent's tmux pane). The discovery stream the interface also consumes is metadata-only (who exists), so this dedicated poll owns process-liveness: the dot turns grey within one poll interval when a process dies, including when the OOM handler (earlyoom) sheds it. The same polled state gates the chat's activity indicator, so a shed or exited agent stops showing "Thinking...". Where several states share a color (RUNNING/RUNNING_UNKNOWN_AGENT_TYPE are both green; DONE/STOPPED/REPLACED/UNKNOWN are all grey), the dot also takes a subtly different circular shape -- a solid disc for the expected state of each color (RUNNING, WAITING, DONE), a hollow ring for a process whose agent type is unconfirmed (RUNNING_UNKNOWN_AGENT_TYPE) or whose tmux window is gone (STOPPED), a ring with a center dot for a superseded process (REPLACED), and a faded disc for an indeterminate one (UNKNOWN) -- so the states stay tellable apart at a glance without relying on the tooltip.
