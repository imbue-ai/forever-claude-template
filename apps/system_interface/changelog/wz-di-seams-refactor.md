- Internal refactor (no user-visible behavior change): removed the test-only seams from the system-interface app factory so its production signature reads as if there were no tests. `create_application` no longer takes `agent_manager=`, `claude_auth_service=`, or `welcome_resender=`; it unconditionally builds and starts the live `AgentManager`. A test that needs a seeded manager builds the app and overwrites `state_of(app).agent_manager` (the broadcaster follows automatically), and auth tests overwrite `state_of(app).claude_auth_service` / `.welcome_resender`. "Don't spawn `mngr observe` in tests" stays handled by `conftest`'s `AgentManager.start` no-op.

- `SystemInterfaceState.broadcaster` is now derived from `agent_manager` (a property returning `agent_manager.broadcaster`) rather than a stored field, so the manager and its broadcaster can never diverge and a single test seed repoints both. The `is_agent_manager_owned` flag is gone; the app always owns the manager it builds, so `shutdown()` always stops it.

- Agent messaging: the `discover`/`send` collaborators that were keyword-only defaulted params on the private `_send_message_to_agent` now live on a small `MngrMessenger` object that `AgentManager` owns. Production calls `messenger.send_to_agent(...)`; tests construct `MngrMessenger(discover=..., send=...)` directly. The `send_message` free function and `_send_message_to_agent` are gone.
