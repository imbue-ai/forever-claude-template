Fix agent-terminal and persistent-terminal tabs intermittently failing to reflow when the split sash is dragged right after switching to the tab by its header (without focusing the terminal body).

Root cause: the ttyd web client's only refit trigger (window resize -> fitAddon.fit) is disposed on every terminal-websocket close and only re-registered when the reconnect completes. Terminal websockets in the desktop client drop intermittently (observed close code 1006 through the forward path), so a sash drag landing inside a reconnect window was silently lost: the terminal kept its stale column count, the reconnect handshake then set the new PTY to that stale size, and nothing re-measured until the user resized again after focusing the terminal.

The dockview host now refits terminal iframes itself (a ResizeObserver on the panel container plus a visibility-change safety net, with short trailing re-checks), calling the ttyd client's `term.fit()` directly. `fit()` is pure client-side geometry, so this works even while the socket is down, and the corrected size reaches the PTY either immediately (socket open) or via the reconnect handshake. Wired for both terminal render paths (agent terminals and persistent session terminals); non-terminal iframes are untouched.
