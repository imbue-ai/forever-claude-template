Revealing a merged system-interface change is now a single self-healing command, `reveal_system_interface.py --rollback-to <pre-merge-sha>`. It classifies what changed and does only what is needed: refreshes dependencies when a manifest changed (`npm ci` / `uv tool install -e apps/system_interface --reinstall`, which a plain restart would otherwise skip), rebuilds the bundle and broadcasts a `reload_system_interface` op (frontend), and/or restarts the services agent (backend). A backend change is pre-flighted on a throwaway port before the live service is touched, then the loopback endpoint is polled to confirm health. If any step fails, the change is automatically rolled back to the known-good revision (as a forward revert commit), rebuilt/restarted from it, and re-confirmed healthy -- so a failed reveal can never leave the user without a working UI. This replaces the separate `reload_system_interface.py` reload-only script and the previous auto-reveal machinery (the conditional-rebuild service step, the `/api/build-id` endpoint and build-id meta tag, and the websocket-reconnect build-id check), which are gone.
