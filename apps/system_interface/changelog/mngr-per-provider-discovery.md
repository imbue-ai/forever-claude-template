- Migrated the system_interface agent manager to mngr's per-provider discovery. It now consumes `ProviderDiscoverySnapshotEvent` (the new per-provider snapshots) instead of the deprecated global `FullDiscoverySnapshotEvent`, folding every discovery event from `mngr observe --discovery-only` into the shared `DiscoveryStateAggregator` and reacting to the returned membership delta. This mirrors the other mngr-side consumers, so a slow or errored provider's snapshot can no longer wipe another provider's agents from the desktop client's agent list. The per-agent app watcher, activity tracking, and assist auto-open are now driven off the aggregator's added/removed deltas, and an already-tracked agent's activity indicator is preserved across snapshots. The `mngr observe` argv is unchanged.
