The workspace UI now reconnects its live-updates WebSocket when the user's machine wakes from sleep (or the window regains focus, or the network comes back). The socket rides an SSH tunnel that dies during sleep without the browser ever firing `onclose`, so the old `onclose`-driven reconnect never ran and the UI silently showed pre-sleep state forever -- events that occurred overnight never appeared until a manual reload. The UI now listens for visibilitychange/focus/online, coalesces the burst a single wake fires into one reconnect, detaches the dead socket's handlers before closing it so its late `onclose` cannot clobber the replacement, and opens a fresh connection whose replayed snapshot brings the view current. On reconnect the local proto-agent set is also reset from the replayed snapshot, so an agent build that finished while disconnected no longer strands a panel on "Creating agent...".
