Tearing down a system-interface preview (`reveal_system_interface.py unpreview`, which also runs as part of `preview` and `reveal`) no longer risks restarting the entire workspace container. Previously, teardown killed each detached preview server's process group by shelling out to `kill -TERM -<pid>`. The external procps-ng `kill` mis-parses a bare negative-pid argument that is not preceded by `--` (procps-ng issue #65) and could deliver SIGTERM to PID 1 instead of the intended process group; because the container's init traps SIGTERM and exits, that turned into an `unless-stopped` restart of the whole container -- killing every agent in the workspace, including the one that invoked the command. Teardown now signals the process group via `os.killpg` directly, so it targets only the intended group and can never escape to PID 1.
