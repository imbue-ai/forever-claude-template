Scheduled agents (e.g. the nightly Caretaker) now surface in the workspace UI. An agent carrying a non-empty `highlight` label gets its own chat tab opened automatically in the main chat group (without stealing focus) and blinks yellow until viewed; bumping the label's value re-blinks the tab for each new run, whether it was left open or closed. Surfacing is driven by one persisted signal -- the highlight key the user last acknowledged (by viewing or closing the tab) versus the agent's current key -- so it is idempotent across WebSocket reconnects. Agent labels are read straight from the discovery stream, so the `highlight` run-key and `is_primary` hiding are always current.

The live-updates WebSocket now reconnects when the machine wakes from sleep, the window refocuses, or the network returns. On sleep the SSH-tunneled socket dies without a browser close event (a phantom "OPEN" state), so the close-driven auto-reconnect never ran and overnight runs stayed invisible until a manual reload. The reconnect also rebuilds the local proto-agent set from the fresh snapshot (completions are not replayed), so an agent that finished building while disconnected no longer strands a tab on "Creating agent...".

Fixed: a fresh mind with no Claude credentials could deadlock on "No events yet" with no way to sign in. The chat panel now backstops the login modal: when a snapshot loads with zero transcript events, it probes `GET /api/claude-auth/status` once per page load and opens the sign-in modal if logged out; after sign-in the existing auth-success resend delivers the welcome.
