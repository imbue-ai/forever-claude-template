Scheduled agents (e.g. the weekly Caretaker) now surface in the workspace UI. An agent carrying a non-empty `highlight` label gets its own chat tab opened automatically in the main chat group (without stealing focus) whenever the label's value changes during the session -- so each new run re-opens the tab if it had been closed, while a page reload never resurrects tabs for runs that predate the page. The last-seen keys live in module state, so a run that fired during a dead-socket gap (a sleeping laptop) still surfaces on the first snapshot after the WebSocket reconnects. Agent labels are read straight from the discovery stream, so the `highlight` run-key and `is_primary` hiding are always current.

The live-updates WebSocket now reconnects when the machine wakes from sleep, the window refocuses, or the network returns. On sleep the SSH-tunneled socket dies without a browser close event (a phantom "OPEN" state), so the close-driven auto-reconnect never ran and overnight runs stayed invisible until a manual reload. The reconnect also rebuilds the local proto-agent set from the fresh snapshot (completions are not replayed), so an agent that finished building while disconnected no longer strands a tab on "Creating agent...".
