- Queued-message "interrupt and send": triggering this action on several queued messages at once no longer fails with "0 successful agents". Each queued bubble has its own "interrupt and send" button, and previously clicking more than one fired overlapping restarts of the same agent that raced each other. The action is now guarded per agent so at most one interrupt-and-resend runs for an agent at a time; while one is in flight, a second bubble's click is ignored (the restart clears the agent's queue anyway).
