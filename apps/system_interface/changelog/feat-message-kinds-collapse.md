Automated, non-human `user_message`s now render as a collapsed system chip instead of a bare user bubble, and the classification behind that is unified into one place.

**New:** a browser-fleet nudge ("Browser foo-1 was handed back to you.") and a background `<task-notification>` line no longer show as if the human typed them. They fold into the current turn as a collapsed, expandable chip -- exactly like Stop-hook feedback -- labelled "Browser fleet" / "Background task".

**Refactor:** the scattered per-message string predicates are replaced by a single classifier. `message-kinds.ts` enumerates every display kind (`UserPrompt`, `SystemChip`, `SkillExpansion`, `Hidden`, `PermissionResolution`) with, for each, its rail and exact net visual -- the one file a new harness reads to see what it must produce. `classifyUserMessage` (in message-classification.ts) is now the only place that maps raw content to a kind; grouping and rendering key off the kind alone and never re-sniff text. Per-kind rendering moved into `user-message-display.ts`.

**General `isMeta` hiding:** Claude Code marks framework-injected, model-only user messages with `isMeta` -- the resume-continuation marker, the image coordinate note ("[Image: ... Multiply coordinates by ...]"), MCP-resource dumps, hook context, and more. The session parser now passes that flag through as `is_meta`, and the classifier hides any `is_meta` message (`UserMessageKind.Hidden`) unless an explicit detector surfaces it. This one rule replaces the bespoke resume-continuation matcher (now removed) and catches the whole family -- present and future -- instead of a detector per message. The override matters: Stop-hook feedback is itself `isMeta` yet is deliberately shown as a chip, so explicit detectors win over the `isMeta` rule.

The split is deliberate: "how a message looks" (kind) and "which harness marker identifies it" (the detector table) are now separate, so adding Codex or another harness means adding detectors that map its markers to these existing kinds -- no change to grouping or rendering. This is display-only: a fleet nudge still starts the agent's next turn (it folds in as a chip, same as a stop hook), and nothing about delivery changes.
