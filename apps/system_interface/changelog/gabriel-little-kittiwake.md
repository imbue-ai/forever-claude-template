- Mobile design for the workspace UI. On phone-sized screens (<=768px) the dockview tab strip -- tiny targets, wheel-driven overflow scrolling, unreachable overflowing tabs on touch -- is replaced by a mobile tab bar: a hamburger button at the top left opens a single bottom-sheet menu holding the open tabs (tap to switch, per-tab close/destroy) followed by the same "open new" items as the desktop add-tab dropdown, with the active tab's title shown beside the button. The app now sizes itself with dynamic viewport units (100dvh) instead of 100vh, so the composer no longer hides behind the mobile browser's retractable URL bar, and a visualViewport sync clamps the app above the on-screen keyboard on iOS Safari (where dvh does not shrink for the keyboard). Text inputs are 16px on small screens, below which iOS zooms the page on focus -- the zoom that used to shove the composer off screen. Transcript/composer padding compacts, dialogs size to the phone, and safe-area insets are respected (viewport-fit=cover).
- Fixed the dockview theme overrides silently not applying: dockview-core injects its stylesheet at runtime after the app bundle and re-applies the theme class on an inner element, so the app's `.dockview-theme-light` variable overrides (tab bar background, active-tab colors, drag-over accent) lost on cascade order and had been rendering dockview's defaults. The overrides are now anchored on the app-owned container class and actually take effect.
- Refactored the frontend stylesheet: the 3100-line style.css is split into styles/{theme,chat,composer,dockview,modals,responsive}.css with style.css as the import hub, and the empty-state overlay/dropdown JS inline styles moved into CSS classes.
