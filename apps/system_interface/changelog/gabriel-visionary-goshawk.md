
- Chat optimistic messages: a "queued" message bubble is now cleared automatically once its agent returns to idle. Previously, if a queued message could never reconcile against the real transcript -- because the user edited it in the Claude terminal before submitting, or the agent was restarted and dropped its queue -- the optimistic bubble stayed up forever. The agent going from a working state to idle is treated as the signal that its queue has drained, so any still-"queued" bubble is dropped. A just-sent message (still "sending", or sent to an already-idle agent) is left untouched, so this does not interfere with the normal send or the "interrupt and send" action.
